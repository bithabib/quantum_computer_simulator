### The research paper (in progress)
- **Venue:** IEEE Access (open access; APC on acceptance). Template: IEEEtran journal.
- **Research idea (FINAL):** *Predict a variational quantum circuit's trainability
  (barren-plateau severity) from its architecture, using a classical ML model
  trained on data generated by the qsim simulator.*
  - Input X = circuit spec (qubits, layers, ansatz, entanglement pattern, entangler
    gate, cost locality, derived counts).
  - Output Y = `log10 Var[dC/dtheta]` (regression) and `barren`/`trainable`
    (classification).
  - The browser/simulator is only the DATA-GENERATION tool — NOT the novelty.
- **Docs:** `paper/research-idea.md` (the idea), `paper/literature-review.md`
  (full cited survey — 28 sources), `paper/outline.md` (IEEE section structure).
- **Key framing rule:** do NOT claim quantum advantage; frame honestly (classical
  ML *about* quantum circuits). Recent skeptical literature (Bowles/Schuld 2024;
  Cerezo *Nat. Commun.* 2025) demands this.

### The ML pipeline (`qml_bp/`)
- `ansatz.py` — random circuit spec → parameter-shift gradient-variance label (uses qsim).
- `generate.py` — parallel dataset generator (multiprocessing, streaming CSV).
- `train.py` — regression + classification, **extrapolation split** (train small
  qubits → test larger), permutation feature importance.
- `README.md` — macOS/M4 run commands. Deps: `qml_bp/requirements.txt`
  (numpy, pandas, scikit-learn).
- **Validated:** labels behave correctly — global-cost circuits are far more barren
  (mean log10 var ≈ −5.5) than local-cost (≈ −2.3), matching theory.

## How to run on the target machine (Ubuntu / Linux — or Mac)
```bash
git clone https://github.com/bithabib/quantum_computer_simulator.git
cd quantum_computer_simulator
python3 -m venv .venv && source .venv/bin/activate
pip install -r qml_bp/requirements.txt
# sanity:
python -m qml_bp.generate --n-specs 300 --samples 80 --qubit-max 8 --out data_bp/sample.csv
python -m qml_bp.train --data data_bp/sample.csv
# big run — set --workers to the core count:
python -m qml_bp.generate --n-specs 100000 --samples 200 --qubit-max 12 \
  --workers "$(nproc)" --out data_bp/bp_dataset.csv
python -m qml_bp.train --data data_bp/bp_dataset.csv
```
On Ubuntu, `--workers "$(nproc)"` uses all cores. A big box will finish a 100k-row
set in well under an hour.
